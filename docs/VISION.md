# Cawnex — Vision

> Coordinated Intelligence. From vision to velocity.

---

## What is Cawnex?

Cawnex is a multi-agent AI orchestration platform that turns a founder's product vision into shipped, tested, documented code — autonomously.

You describe what you want to build. Cawnex breaks it into milestones, generates tasks, assigns specialized AI agents ("Crows"), and delivers merged pull requests. You stay in control — every task needs your approval before execution.

**One sentence:** Vision → Milestones → Tasks → Crows → PRs → Shipped code.

---

## Who is it for?

**Technical founder CEOs** who:
- Have a clear product vision but limited time to code everything themselves
- Want AI to handle the execution while they steer the direction
- Need to ship fast without hiring a team
- Want to maintain quality through automated review and human approval gates

---

## Core Principles

1. **Vision-driven** — Everything flows from the project vision. The AI understands your intent and works toward it.
2. **Human-in-the-loop, not human-in-the-way** — AI proposes, you approve. One approval point per task, not death by a thousand confirmations.
3. **BYOL (Bring Your Own LLM)** — You provide your own API keys. Cawnex never pays for tokens, never throttles your usage, never sees your prompts. ~95% gross margin.
4. **Multi-tenant from day 1** — Safe data isolation between tenants at every layer.
5. **Orchestration is the product** — Anyone can spawn an agent. The value is coordinating them: routing, sequencing, context-sharing, synchronized PRs.
6. **Full native mobile** — iOS (Swift/SwiftUI) and Android (Kotlin/Compose). No cross-platform compromises.
7. **Observable by default** — Every action is streamed. Every cost is tracked. Every decision is auditable.

## Glossary

| Term | Meaning |
|------|---------|
| **Crow** | A specialized AI agent (e.g., Refine Crow, Dev Crow, QA Crow, Docs Crow) |
| **The Murder** | The orchestrator that coordinates all Crows across a workflow |
| **Nest** | An isolated git worktree where a Crow works |
| **The Roost** | The dashboard / command center (iOS app + web) |
| **Project** | A top-level container with its own vision, milestones, repos, workflow, and crow config |
| **Vision** | A conversational AI-refined description of what the project should become |
| **Milestone** | A significant checkpoint toward the vision, containing multiple tasks |
| **Task** | A unit of work generated by AI from milestones. Requires human approval before execution |
| **Workflow** | A configurable pipeline of steps (e.g., Refine → Implement → Review → Document) |
| **Execution** | A single run of a workflow against a task |
| **Tenant** | A company or user account. All data is isolated per tenant |
| **BYOL** | Bring Your Own LLM — users provide their own API keys |
| **.pen** | Pencil's JSON-based design file format, used for design-as-code |

## The Flow

```
┌─────────────────────────────────────────────────────────┐
│                    FOUNDER (iOS App)                      │
│                                                           │
│  1. Create Project                                        │
│  2. Chat with Vision AI → define what to build            │
│  3. AI proposes Milestones → approve/edit/reject          │
│  4. AI generates Tasks per milestone → approve/edit       │
│  5. Watch Crows execute in real-time                      │
│  6. Review PRs → merge from phone                         │
│  7. Milestone complete → AI proposes next tasks           │
└───────────────────────────┬─────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────┐
│                    THE MURDER (Backend)                    │
│                                                           │
│  Receives approved task                                   │
│  → Selects workflow (Refine → Implement → Review → Doc)   │
│  → Assigns Crows to each step                             │
│  → Creates Nest (git worktree) per Crow                   │
│  → Streams events to iOS app (SSE)                        │
│  → Opens PR when implementation is done                   │
│  → QA Crow reviews → auto-merge or request changes        │
│  → Docs Crow updates documentation                        │
│  → Reports cost, duration, outcome                        │
└─────────────────────────────────────────────────────────┘
```

### Detailed Task Lifecycle

```
AI generates task (from milestone)
    ↓
PENDING APPROVAL → push notification to founder
    ↓
Founder approves (or edits + approves, or rejects)
    ↓
QUEUED → enters SQS
    ↓
Murder picks up → selects workflow → starts execution
    ↓
REFINING → Refine Crow creates user story + acceptance criteria
    ↓
IMPLEMENTING → Dev Crow writes code in Nest, opens PR
    ↓
REVIEWING → QA Crow reviews PR against acceptance criteria
    ↓
DOCUMENTING → Docs Crow updates docs (optional, skippable)
    ↓
COMPLETED → PR merged, execution recorded
```

## What Cawnex is NOT

- **Not a code editor** — we don't compete with Cursor or Copilot
- **Not a single-agent builder** — we orchestrate multiple specialized agents
- **Not an LLM provider** — we never host models, users bring their own
- **Not a project management tool** — GitHub Issues is the source of truth, we're the execution layer
